# سامانه ارسال رمز یک‌بار مصرف (OTP)

سامانه ارسال رمز یکبار مصرف (OTP) از طریق بله راهکاری سریع، امن و مقرون‌به‌صرفه برای کسب‌وکارهایی است که به احراز هویت کاربران خود نیاز دارند.
این سرویس امکان ارسال مستقیم OTP از طریق بله را فراهم می‌کند و جایگزینی مناسب برای پیامک است. با استفاده از این روش، می‌توانید رمزهای یکبار مصرف را سریع‌تر، مطمئن‌تر برای مشتریان خود ارسال کنید.

برای ثبت‌نام و تهیه اشتراک جهت استفاده از این سامانه، ابتدا باید وارد سامانه [درگاه بله (opens in a new tab)](https://safir.bale.ai/gateway) شوید.
پس از تهیه اشتراک و دریافت رمز عبور و نام کاربری، قادر خواهید بود از سرویس‌های ارسال رمز یکبار مصرف استفاده کنید.

* * *

## سرویس احراز هویت [Permalink for this section](https://docs.bale.ai/gateway\#%D8%B3%D8%B1%D9%88%DB%8C%D8%B3-%D8%A7%D8%AD%D8%B1%D8%A7%D8%B2-%D9%87%D9%88%DB%8C%D8%AA)

پس از دریافت رمز عبور و نام کاربری، باید از طریق سرویس احراز هویت سامانه، **یک توکن JWT** دریافت کنید.
این توکن برای تایید هویت شما استفاده شده و در ادامه برای ارسال پیام‌های رمز یکبار مصرف به کاربران به‌کار می‌رود. با این روش، فرآیند ارسال رمز یکبار مصرف به‌صورت ایمن و معتبر انجام خواهد شد.

#### آدرس API: `/auth/token` [Permalink for this section](https://docs.bale.ai/gateway\#%D8%A2%D8%AF%D8%B1%D8%B3-api-authtoken)

#### متد: `POST` [Permalink for this section](https://docs.bale.ai/gateway\#%D9%85%D8%AA%D8%AF-post)

این API به کلاینت‌ها این امکان را می‌دهد که با استفاده از **اعتبارسنجی کلاینت** (شناسه کلاینت و رمز عبور کلاینت) احراز هویت کنند و توکن دسترسی برای درخواست‌های بعدی دریافت کنند.
پارامتر های بدنه این درخواست باید به شکل form-data به سمت سرور ارسال شوند.

#### URL: [Permalink for this section](https://docs.bale.ai/gateway\#url)

```
POST https://safir.bale.ai/api/v2/auth/token
```

#### هدرها [Permalink for this section](https://docs.bale.ai/gateway\#%D9%87%D8%AF%D8%B1%D9%87%D8%A7)

- Content-Type: `application/x-www-form-urlencoded`

#### curl [Permalink for this section](https://docs.bale.ai/gateway\#curl)

```
curl -X POST "https://safir.bale.ai/api/v2/auth/token" \
     -d "grant_type=client_credentials" \
     -d "client_secret={password}" \
     -d "scope=read" \
     -d "client_id={username}"
```

#### پارامترهای بدنه درخواست: [Permalink for this section](https://docs.bale.ai/gateway\#%D9%BE%D8%A7%D8%B1%D8%A7%D9%85%D8%AA%D8%B1%D9%87%D8%A7%DB%8C-%D8%A8%D8%AF%D9%86%D9%87-%D8%AF%D8%B1%D8%AE%D9%88%D8%A7%D8%B3%D8%AA)

| پارامتر | نوع | توضیحات | مثال |
| --- | --- | --- | --- |
| `grant_type` | رشته | نوع گرانت درخواست شده. باید برابر با `client_credentials` باشد. | `client_credentials` |
| `client_secret` | رشته | کلید رمز عبور مرتبط با شناسه کلاینت شما. | `password` |
| `scope` | رشته | سطح دسترسی درخواست شده. برای این درخواست باید مقدار آن `read` باشد. | `read` |
| `client_id` | رشته | شناسه کلاینت شما. | `username` |

### پاسخ [Permalink for this section](https://docs.bale.ai/gateway\#%D9%BE%D8%A7%D8%B3%D8%AE)

#### موفقیت آمیز: [Permalink for this section](https://docs.bale.ai/gateway\#%D9%85%D9%88%D9%81%D9%82%DB%8C%D8%AA-%D8%A2%D9%85%DB%8C%D8%B2)

در صورت موفقیت در احراز هویت، پاسخ شامل اطلاعات زیر خواهد بود:

| پارامتر | نوع | توضیحات | مثال |
| --- | --- | --- | --- |
| `access_token` | رشته | توکن دسترسی که برای انجام درخواست‌های بعدی باید استفاده شود. | `e..f` |
| `expires_in` | عدد | زمان انقضای توکن (بر حسب ثانیه). | `43200` |
| `scope` | رشته | سطح دسترسی درخواست شده. | `read` |
| `token_type` | رشته | نوع توکن. | `bearer` |

### خطاها [Permalink for this section](https://docs.bale.ai/gateway\#%D8%AE%D8%B7%D8%A7%D9%87%D8%A7)

در صورت وجود مشکلی در درخواست یا احراز هویت، پاسخ با کد وضعیت مناسب و پیامی خطا همراه خواهد بود.

#### ۱. خطای احراز هویت نامعتبر: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B1-%D8%AE%D8%B7%D8%A7%DB%8C-%D8%A7%D8%AD%D8%B1%D8%A7%D8%B2-%D9%87%D9%88%DB%8C%D8%AA-%D9%86%D8%A7%D9%85%D8%B9%D8%AA%D8%A8%D8%B1)

در صورتی که شناسه کلاینت یا رمز عبور اشتباه باشد، پاسخ به صورت زیر خواهد بود:

```
{
 "error": "invalid_client",
 "error_description": "Client authentication failed"
}
```

کد وضعیت: `401 Unauthorized`

#### ۲. خطای پارامترهای ناقص یا اشتباه: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B2-%D8%AE%D8%B7%D8%A7%DB%8C-%D9%BE%D8%A7%D8%B1%D8%A7%D9%85%D8%AA%D8%B1%D9%87%D8%A7%DB%8C-%D9%86%D8%A7%D9%82%D8%B5-%DB%8C%D8%A7-%D8%A7%D8%B4%D8%AA%D8%A8%D8%A7%D9%87)

اگر برخی از پارامترهای ضروری درخواست ارسال نشده باشند یا اشتباه باشند، پاسخ به صورت زیر خواهد بود:

```
"Failed to Parse Request"
```

کد وضعیت: `400 Bad Request`

#### ۳. خطای سرور: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B3-%D8%AE%D8%B7%D8%A7%DB%8C-%D8%B3%D8%B1%D9%88%D8%B1)

در صورت بروز مشکل داخلی در سرور، پاسخ به صورت زیر خواهد بود:

```
{"type": 1, "code": 2, "message": "internal server error occurred"}
```

کد وضعیت: `500 Internal Server Error`

### نکات مهم [Permalink for this section](https://docs.bale.ai/gateway\#%D9%86%DA%A9%D8%A7%D8%AA-%D9%85%D9%87%D9%85)

- برای دریافت توکن دسترسی، حتماً باید شناسه کلاینت (`client_id`) و رمز عبور کلاینت (`client_secret`) معتبر و صحیح باشند.
- توکن دسترسی که دریافت می‌کنید تنها برای مدت زمان مشخصی معتبر است و پس از آن باید دوباره درخواست توکن جدید ارسال کنید.

## سرویس ارسال رمز یکبار مصرف [Permalink for this section](https://docs.bale.ai/gateway\#%D8%B3%D8%B1%D9%88%DB%8C%D8%B3-%D8%A7%D8%B1%D8%B3%D8%A7%D9%84-%D8%B1%D9%85%D8%B2-%DB%8C%DA%A9%D8%A8%D8%A7%D8%B1-%D9%85%D8%B5%D8%B1%D9%81)

#### آدرس API: `/send_otp` [Permalink for this section](https://docs.bale.ai/gateway\#%D8%A2%D8%AF%D8%B1%D8%B3-api-send_otp)

این API به شما این امکان را می‌دهد که یک رمز یکبار مصرف (OTP) به شماره تلفن کاربر ارسال کنید.
برای استفاده از این سرویس، ابتدا باید از طریق سرویس احراز هویت توکن دسترسی معتبر دریافت کرده باشید و سپس این توکن را در هدر درخواست ارسال کنید.

#### متد: `POST` [Permalink for this section](https://docs.bale.ai/gateway\#%D9%85%D8%AA%D8%AF-post-1)

این API به کلاینت‌ها این امکان را می‌دهد که با استفاده از **اعتبارسنجی کلاینت** (شناسه کلاینت و رمز عبور کلاینت) احراز هویت کنند و توکن دسترسی برای درخواست‌های بعدی دریافت کنند.
پارامتر های بدنه این درخواست باید به شکل form-data به سمت سرور ارسال شوند.

### درخواست [Permalink for this section](https://docs.bale.ai/gateway\#%D8%AF%D8%B1%D8%AE%D9%88%D8%A7%D8%B3%D8%AA)

#### URL: [Permalink for this section](https://docs.bale.ai/gateway\#url-1)

```
POST https://safir.bale.ai/api/v2/send_otp
```

#### هدرها [Permalink for this section](https://docs.bale.ai/gateway\#%D9%87%D8%AF%D8%B1%D9%87%D8%A7-1)

  (توکن دسترسی باید در هدر درخواست ارسال شود.)

#### curl [Permalink for this section](https://docs.bale.ai/gateway\#curl-1)

```
curl -X POST "https://safir.bale.ai/api/v2/send_otp" \

     -H "Content-Type: application/json" \
     -d '{
           "phone": "989123456789",
           "otp": 111111
         }'
```

#### پارامترهای بدنه درخواست: [Permalink for this section](https://docs.bale.ai/gateway\#%D9%BE%D8%A7%D8%B1%D8%A7%D9%85%D8%AA%D8%B1%D9%87%D8%A7%DB%8C-%D8%A8%D8%AF%D9%86%D9%87-%D8%AF%D8%B1%D8%AE%D9%88%D8%A7%D8%B3%D8%AA-1)

| پارامتر | نوع | توضیحات | مثال |
| --- | --- | --- | --- |
| `phone` | رشته | شماره تلفن همراه کاربر (در فرمت بین‌المللی) | `989123456789` |
| `otp` | عدد | رمز یکبار مصرف (OTP) که باید ارسال شود. (عدد 3 تا 8 رقمی) | `111111` |

#### نمونه درخواست: [Permalink for this section](https://docs.bale.ai/gateway\#%D9%86%D9%85%D9%88%D9%86%D9%87-%D8%AF%D8%B1%D8%AE%D9%88%D8%A7%D8%B3%D8%AA)

```
{
  "phone": "989123456789",
  "otp": 111111
}
```

شماره تماس باید طبق فرمت خاصی وارد شود. شماره تلفن‌های معتبر باید با پیش‌شماره 98 شروع شده و بدون صفر نوشته شوند. به این ترتیب، شماره‌های موبایل ایرانی که معمولاً با صفر شروع می‌شوند، باید ابتدا صفر حذف شود و سپس با کد کشور ایران (98) جایگزین شود.
این قاعده به شما کمک می‌کند که شماره تلفن‌ها به درستی پردازش شوند. در جدول زیر، برخی از شماره‌ها آورده شده و صحت آن‌ها بررسی شده است:

| شماره ورودی | صحیح یا غلط |
| --- | --- |
| `09123456789` | غلط |
| `9123456789` | غلط |
| `+980912345678` | غلط |
| `989123456789` | صحیح |

همان‌طور که مشاهده می‌کنید، شماره‌ها باید بدون صفر ابتدایی و با پیش‌شماره 98 آغاز شوند تا معتبر باشند.

### خطاها [Permalink for this section](https://docs.bale.ai/gateway\#%D8%AE%D8%B7%D8%A7%D9%87%D8%A7-1)

#### ۱. خطای شماره تلفن نامعتبر: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B1-%D8%AE%D8%B7%D8%A7%DB%8C-%D8%B4%D9%85%D8%A7%D8%B1%D9%87-%D8%AA%D9%84%D9%81%D9%86-%D9%86%D8%A7%D9%85%D8%B9%D8%AA%D8%A8%D8%B1)

اگر شماره تلفن وارد شده نامعتبر باشد، پاسخ به صورت زیر خواهد بود:

#### ۲. کاربر پیدا نشد: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B2--%DA%A9%D8%A7%D8%B1%D8%A8%D8%B1-%D9%BE%DB%8C%D8%AF%D8%A7-%D9%86%D8%B4%D8%AF)

اگر کاربر با شماره تلفن وارد شده در سیستم پیدا نشود، پاسخ به صورت زیر خواهد بود:

```
{"type": 3, "code": 17, "message": "this phone does not have an account in Bale"}
```

کد وضعیت: `404 Not Found`

#### ۳. خطای عدم موجودی کافی: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B3-%D8%AE%D8%B7%D8%A7%DB%8C-%D8%B9%D8%AF%D9%85-%D9%85%D9%88%D8%AC%D9%88%D8%AF%DB%8C-%DA%A9%D8%A7%D9%81%DB%8C)

### محدودیت نرخ (Rate Limit) [Permalink for this section](https://docs.bale.ai/gateway\#%D9%85%D8%AD%D8%AF%D9%88%D8%AF%DB%8C%D8%AA-%D9%86%D8%B1%D8%AE-rate-limit)

#### ۱. محدودیت نرخ برای هر شماره تلفن: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B1-%D9%85%D8%AD%D8%AF%D9%88%D8%AF%DB%8C%D8%AA-%D9%86%D8%B1%D8%AE-%D8%A8%D8%B1%D8%A7%DB%8C-%D9%87%D8%B1-%D8%B4%D9%85%D8%A7%D8%B1%D9%87-%D8%AA%D9%84%D9%81%D9%86)

- **حداکثر ۳۰ درخواست در ساعت** مجاز است.

- اگر تعداد درخواست‌های ارسالی به یک شماره تلفن در مدت یک ساعت بیش از ۳۰ بار شود، در ادامه امکان ارسال رمز یکبار مصرف برای آن شماره میسر نخواهد بود.

- در این صورت، درخواست‌های بعدی با پاسخ زیر مواجه خواهند شد:


```
{"type": 2, "code": 18, "message": "rate limit exceeded"}
```

#### ۲. محدودیت کلی برای هر سازمان: [Permalink for this section](https://docs.bale.ai/gateway\#%DB%B2-%D9%85%D8%AD%D8%AF%D9%88%D8%AF%DB%8C%D8%AA-%DA%A9%D9%84%DB%8C-%D8%A8%D8%B1%D8%A7%DB%8C-%D9%87%D8%B1-%D8%B3%D8%A7%D8%B2%D9%85%D8%A7%D9%86)

همچنین برای هر عضو سامانه یک محدودیت کلی هم اعمال شده که موجب میشود هر عضو در دقیقه بتواند ۳۰۰ درخواست ارسال رمز یکبار مصرف ارسال کند.

#### کنترل Burst Rate: [Permalink for this section](https://docs.bale.ai/gateway\#%DA%A9%D9%86%D8%AA%D8%B1%D9%84-burst-rate)

نرخ ارسال پیام‌ها باید شامل **کمترین burst ممکن** باشد.

- این به معنای توزیع یکنواخت درخواست‌ها در بازه‌های زمانی مجاز است تا از ارسال ناگهانی تعداد زیادی درخواست جلوگیری شود.
- در صورتی که درخواست‌ها به‌صورت ناگهانی و در حجم بالا ارسال شوند، حتی اگر در محدوده کلی مجاز باشند، ممکن است به دلیل burst بالا رد شوند.

### نکات مهم [Permalink for this section](https://docs.bale.ai/gateway\#%D9%86%DA%A9%D8%A7%D8%AA-%D9%85%D9%87%D9%85-1)

- برای ارسال OTP، باید توکن دسترسی معتبر (که از سرویس احراز هویت دریافت کرده‌اید) در هدر درخواست ارسال شود.
- کد OTP باید عددی ۳ تا ۸ رقمی و معتبر باشد.
- پس از ارسال OTP، هزینه ارسال از موجودی حساب کسر خواهد شد، بنابراین باید موجودی کافی داشته باشید.
- در صورت موفقیت، پاسخ شامل موجودی باقی‌مانده خواهد بود.
